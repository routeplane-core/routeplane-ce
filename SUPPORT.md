# Support

Routeplane CE is free, Apache-2.0 licensed software maintained by the Routeplane team. It ships
**without an SLA**: we triage issues and answer questions on a best-effort basis, and we are
honest that response times vary with maintainer bandwidth. An active issue tracker matters to us
— you can judge current responsiveness from the tracker itself rather than from promises here.

## Where to go

| Need | Channel |
|---|---|
| Questions, how-do-I, config help | [GitHub Discussions — Q&A](https://github.com/routeplane-core/routeplane-ce/discussions/categories/q-a) |
| Bug reports | [GitHub Issues](https://github.com/routeplane-core/routeplane-ce/issues/new/choose) — please include a reproduction |
| Feature ideas | [GitHub Discussions — Ideas](https://github.com/routeplane-core/routeplane-ce/discussions/categories/ideas) first; maintainers promote accepted ideas to issues |
| Security vulnerabilities | **security@routeplane.ai** — see [SECURITY.md](SECURITY.md); never a public issue |
| Anything else for the maintainers | **maintainers@routeplane.ai** |

## Bug reports need a reproduction

A bug we can reproduce usually gets fixed quickly; one we cannot reproduce usually stalls. The
issue template asks for the gateway version, provider(s) involved, deployment method, a
minimal config, and logs — filling those in is the single biggest thing you can do to get your
bug fixed.

## Commercial support and the enterprise platform

If you need an SLA, sovereign data-residency routing with signed audit artifacts, the MCP
agentic-security gateway, SSO/RBAC, or the rest of the commercial platform, contact
**hello@routeplane.ai**. The exact CE vs. enterprise feature boundary is documented in the
README — CE is not a trial and does not expire.
